The Story

What this phase answers

The Capital Thesis answers a single question: is there fundable money for this type of venture? Before we research problems, design solutions, or write a line of code, we need to know that capital exists and what funders actually require. This is the capital-first principle — start with where the money is, not where the idea is.

What the agents actually do

The process starts when a human provides funder criteria — a description of the funding landscape to investigate. From there, an AI agent reads that input and generates tailored research prompts, each scoped to a specific question with citation requirements baked in. A human then runs those prompts through deep research. The raw results come back, and a processing agent extracts every cited claim, builds structured funder profiles, and writes a narrative thesis with inline citations. A validation agent then runs 42 automated checks. Finally, a gate review agent presents the evidence to a human for a proceed/revise/kill decision.

Why this structure matters

Every step produces auditable files. The processing log records what was included, what was excluded, and why. The validation report checks citation coverage, cross-file consistency, and source quality. The gate review presents strengths and weaknesses — the agent does not advocate; it presents evidence and lets the human decide. Nothing proceeds without a human saying "proceed."

What Phase 01 Produces

Every file is committed to the project repository. Nothing is ephemeral. Anyone can audit the full chain from raw research to final thesis.

thesis.md
Narrative capital thesis with inline citations. Every factual claim linked to a third-party source.
funder-profiles.yaml
Structured data for each funder: criteria, amounts, timelines, viability assessment, alignment gaps.
sources.md
Deduplicated index of every source cited. Org name, document title, date, URL, what it was cited for.
processing-log.md
Audit trail of every processing decision. What was included, excluded, and the reasoning.
validation-report.md
Results of 42 automated checks. Evidence briefing with strengths and weaknesses.
gate-review-record.md
Human decision record. What was decided, what evidence supported it, any revision notes.
03

Why This Structure Works

Three properties that separate this from prompt engineering or "just asking ChatGPT."

Full Audit Trail

Every agent produces log files. The processing log records what evidence was used, what was excluded, and why. The validation report lists every check that passed or failed. You can trace any claim in the thesis back to its original source in the research.

Humans Decide, AI Executes

AI agents do research, synthesis, and validation. But no phase advances without a human explicitly choosing "proceed." The gate review agent presents evidence for and against — it does not advocate. The human sees the weaknesses, not just the strengths.

Repeatable and Inspectable

Every agent is a defined skill with documented inputs, outputs, and procedures. Run the same inputs through the same pipeline and get comparable outputs. No hidden prompts, no one-off conversations that vanish. The skills are version-controlled code.

This same pattern repeats for every phase. Phase 02 (Problem Thesis) uses the same six-step pipeline with different agents: problem-specific prompts, problem-specific processing, problem-specific validation — but the same structure of generate, research, process, validate, gate, present. The framework scales by repeating a proven pattern, not by adding complexity.
